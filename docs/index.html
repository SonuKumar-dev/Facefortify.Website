Getting Started

Welcome to FaceFortify. This utility provides cryptographic protection for your local directories, securing files directly on your machine through biometric facial recognition, hardware binding, and zero-knowledge AES-256 encryption.

First Launch

When launching FaceFortify for the first time on your desktop computer:

  1. The application will prompt you to grant camera permissions. Allow camera access.
  2. You will be directed to the initialization page. Click "Get Started".
  3. Enter your user name. This name is used solely for console display greetings and profile labeling; it is not uploaded to any remote server.
💡
Tip: Input a clean, recognizable profile name. If multiple users register profiles on the same hardware system, this profile identity tracks settings independently.

Face Registration

FaceFortify captures 30 diagnostic frames of your face to create a mathematical face fingerprint. This registration process takes between 30 and 60 seconds.

Registration Best Practices

  • Good Lighting: Face a window or light source. Avoid direct backlighting, which obscures facial coordinates.
  • Positioning: Sit directly in front of the lens. Look straight ahead.
  • Green Box Validation: Keep your head inside the green rectangle displayed on screen.
  • No Obstructions: Temporarily remove hats, sunglasses, or bulky scarves. Normal vision-correction glasses are acceptable.
⚠️
Anti-Spoofing: The ArcFace system maps eye coordinates to detect eye movement. Both eyes must remain visible inside the camera frame. If you wear glasses, make sure glare does not cover your pupils.

Troubleshooting Registration

  • "No face detected": Move slightly closer to the webcam or adjust the screen angle.
  • "No eyes detected": Ensure light isn't creating reflections on your eyeglasses.
✅
Success state: Once the counter hits 30/30 frames, your face fingerprint is compiled into a 512-dimensional vector. The temporary camera images are permanently deleted.

Backup Password

Biometric authentication is highly secure, but changes in physical hardware (motherboard replacement, CPU upgrade) will change your cryptographic salt and break the automatic key derivation pipeline.

Why a Backup Password is Required

To avoid permanent data loss when transferring files between devices or recovering from hardware failure, FaceFortify allows you to establish a strong backup password. This password behaves as a recovery key that can decrypt vaults without hardware bindings or biometric validation.

How to Configure

  1. Navigate to the **Settings** console (gear icon).
  2. Select the **Backup Password** section.
  3. Click **"Set Backup Password"** and input a strong combination.
  4. Store this password in a secure password manager.
⚠️
No Backdoor: There is no master recovery system or cloud password reset. If you lose your computer hardware AND forget your backup password, your encrypted vaults are mathematically unrecoverable.

Locking Folders

Locking is the process of encrypting folder contents and locking OS-level directory control permissions.

How to Lock a Folder

  1. Click "Add Folder" in the dashboard navigation.
  2. Browse your local directory structure and select the target folder.
  3. Verify the path is added to your secure folders list.
  4. Click "Lock All" or click the individual lock icon next to the folder path.
  5. Look at the webcam for a quick 10-frame validation scan.

What Happens Under the Hood

  • Every file inside the directory is sequentially encrypted using a generated AES-256 key.
  • Files are given an encrypted signature structure, and file headers are randomized.
  • OS-level Access Control Lists (ACLs on Windows) or permissions (chmod on Linux/macOS) are updated to restrict reading or writing to the directory.
⚠️
Safety Warning: Do not attempt to lock operating system folders (e.g. C:\Windows or /usr/bin) or directories where applications are currently writing data. This will crash your system or corrupt files.

Unlocking Folders

Unlocking generates your cryptographic key, decrypts the directories, and restores standard access rights.

How to Unlock

  1. Open the FaceFortify dashboard.
  2. Click "Unlock All" or click the unlock button on a specific vault directory.
  3. Align your face in the camera frame. The application will scan 10 frames (~5-10 seconds).
  4. Upon validation, directory permissions are opened and files are decrypted back to their original formats.

If Verification Fails

If you have a beard, poor lighting, or a camera issue:

  1. Click the "Use Backup Password" action on the scanning screen.
  2. Input your recovery password.
  3. Folders will immediately decrypt, bypassing the biometric and hardware binding checks.

Managing Folders

You can manage and monitor your vaults directly from the dashboard listing page.

Removing Directories

If you no longer need biometric shielding on a folder, you can remove it from the tracking list. **Always unlock the folder first before removing it.**

⚠️
Orphaned Vaults: If you remove a folder from the dashboard while it is still in a locked state, it will remain encrypted on your hard drive. You will need to re-add the folder to the dashboard list to decrypt it.

Dashboard Shortcuts

  • Ctrl + L (Cmd + L on Mac): Instantly lock all folders.
  • Ctrl + U (Cmd + U on Mac): Open biometric validation screen to decrypt all.
  • Ctrl + , (Cmd + , on Mac): Open Settings page.

Windows Setup Guide

Installation instructions for Microsoft Windows platforms.

System Requirements

  • Windows 10 or Windows 11 (64-bit architecture)
  • 500 MB free hard drive storage space
  • Webcam (built-in or external USB)
  • Administrator permissions (for initial setup)

Installation Timeline

1
Download the Installer

Download FaceFortify-Setup-1.2.0.exe from the download panel or run the PowerShell terminal script command on the home screen.

💡
Note: Browsers might tag new, unsigned installer files as suspicious. Select "Keep" to save the file.
2
Execute Installer

Double-click the installer file to run it.

⚠️
SmartScreen Bypass: If Windows Defender prompts you with a warning, select "More info" and click "Run anyway". This warning occurs because the installer is unsigned.
3
Setup Wizard

Follow the prompt choices. Check the box to add a desktop shortcut if desired, and click "Install". The installation process takes approximately 30 seconds.

4
Finalize Launch

Click finish to launch FaceFortify. Grant webcam permissions when Windows prompts you.

Uninstallation

Go to Settings → Apps → Installed Apps → FaceFortify → Uninstall. Your settings and encrypted vaults are located in your local user space (%APPDATA%\FaceFortify). If you wish to delete them, you must remove this folder manually.

Linux Setup Guide

Installation instructions for Linux distributions.

System Requirements

  • Modern Linux kernel version 4.0 or newer
  • x86_64 CPU architecture
  • FUSE support (required to run AppImage formats)
  • Camera with Video4Linux2 (V4L2) driver support

Installation Steps

1
Download AppImage

Download the latest FaceFortify-Linux-x86_64.AppImage binary.

2
Change Permissions

Open your command line and run the execute permission command:

chmod +x FaceFortify-Linux-x86_64.AppImage

Alternatively, right-click the file in your desktop GUI manager, go to Properties → Permissions, and check **"Allow executing file as program"**.

3
Execute and Mount

Launch FaceFortify by double-clicking the file or executing it from the terminal shell:

./FaceFortify-Linux-x86_64.AppImage

Camera Access Note

If FaceFortify displays a blank screen or cannot connect to the camera device, verify group access configuration by adding your user profile to the local video group:

sudo usermod -aG video $USER

Then log out and log back in to apply changes.

macOS Setup Guide

Installation instructions for Apple macOS systems.

Requirements & Versions

  • macOS 10.15 (Catalina) or newer
  • Intel Macs: Download FaceFortify-macOS-x64.dmg
  • Apple Silicon (M1/M2/M3) Macs: Download FaceFortify-macOS-arm64.dmg

Installation Steps

1
Mount the DMG

Double-click the downloaded DMG file to open the volume disk.

2
Drag to Applications

Drag the FaceFortify application icon directly into your Applications directory window.

3
Bypass Gatekeeper

Due to the app being unsigned, opening it normally will show a warning saying the developer cannot be verified.

⚠️
Bypass Steps: Go to Applications → Right-click (or Control+click) FaceFortify → Select **"Open"** from the drop-down menu → Click **"Open"** on the confirmation prompt. This is only required on the first launch.
4
Authorize System Permissions

When prompted by macOS, click **"OK"** to grant camera access. If you click deny, open System Settings → Privacy & Security → Camera, and toggle FaceFortify to active status.

Security Architecture Overview

FaceFortify uses a defense-in-depth security approach that requires both biometric authentication and hardware binding validation to generate decryption keys. It is built as a zero-knowledge local utility, meaning no security keys or biometric files are sent to the cloud.

The Defense-in-Depth Layer Stack

Layer Mechanism Mitigates Against
1. Physical User must have device access. Remote network attacks.
2. OS Access Account credentials and privileges. Unauthorized local users on other logins.
3. Biometric ArcFace 512-dimension verification. Stolen account sessions, password leaks.
4. Hardware CPU/Motherboard signature hash binds. Stolen disk drives, copied key files.
5. Cryptography AES-256-CBC with PBKDF2. Brute-force attacks, offline forensics.

Encryption Standards

FaceFortify implements industry-standard cryptography models to guarantee local file integrity.

AES-256-CBC Encryption

Every file within a locked directory is encrypted individually using Advanced Encryption Standard (AES) with a 256-bit key length in Cipher Block Chaining (CBC) mode. This configuration is approved by government agencies for protecting top-secret data.

PBKDF2 Key Derivation

Your biometric vector is not used directly as the decryption key. Instead, the biometric signature is processed through Password-Based Key Derivation Function 2 (PBKDF2):

  • Processes the inputs through 100,000 iterations of HMAC-SHA256.
  • Slows dictionary attack attempts down to 10 guesses per second.
  • Applies a unique salt generated for your specific machine profile.

Biometric Engine

Details of our local, offline facial recognition system.

ArcFace Deep Learning Model

FaceFortify packages a localized deployment of the ArcFace neural network. This network maps facial topology by tracking coordinates, generating a 512-dimensional vector description. This model boasts a 99%+ accuracy rating on public benchmarks and adapts to changes in hair, facial features, aging, and glasses.

Live Eye Tracking

To prevent photo-spoofing attacks (where someone holds up a printout or digital photo of your face to your webcam), the application maps and verifies live eye-blinking and pupil movements. A static image will fail this tracking step and be rejected.

Hardware Bindings

Linking files cryptographically to the host hardware.

How It Works

During registration, FaceFortify queries your system hardware to extract structural details, including:

  1. The CPU hardware serial identifier.
  2. The motherboard manufacturer and board serial number.
  3. The MAC addresses of active network interfaces.

These strings are combined and hashed to generate a unique **Machine ID**. This ID is mixed into the PBKDF2 key derivation pipeline as a key salt. Because of this binding, copying your encrypted files to another computer will fail decryption, even if an attacker has your biometric signature.

Privacy & Storage

FaceFortify is built with privacy in mind. No telemetry or analytics modules are built into the app's packages.

File System Storage Locations

Settings, biometric parameters, and locked registry files are stored in your user configuration directories:

  • Windows: C:\Users\[Username]\AppData\Roaming\FaceFortify
  • Linux: ~/.local/share/FaceFortify
  • macOS: ~/Library/Application Support/FaceFortify

Database Profiles

  • face.dat: The compiled mathematical face representation (512 numbers).
  • keyinfo.dat: Key derivation salts and verification hashes.
  • folders.json: A list tracking which folders are locked.

Threat Matrix

Understanding what FaceFortify protects against and its security limitations.

Protected Risks

  • Laptop Loss or Theft: Stolen drives remain encrypted and unreadable without your biometrics and hardware.
  • Disk Removal: Removing the hard drive and connecting it to another system fails decryption because the hardware binding signatures won't match.
  • Shared Computer Access: Unauthorized family members, colleagues, or roommates cannot access your locked vaults.

Out of Scope Risks

  • System Keyloggers: Active malware with keylogger access can intercept backup passwords if typed.
  • Root/Admin Access Malware: If a virus gains root-level permissions on your active session, it can dump memory hashes or capture video streams.
  • Physical Coercion: The application has no defense if you are physically forced to unlock your device.

Backup & Recovery

How to recover your files if your system fails or your biometrics change.

Recovery Scenarios

Scenario A: System hardware fails (motherboard or CPU dies)

  • Install FaceFortify on your new replacement computer.
  • Launch the recovery prompt.
  • Input your **Backup Password** to decrypt and restore your vaults.
⚠️
Important: If you do not configure a backup password, files on a failed motherboard or CPU cannot be decrypted on a new system. Take 5 minutes to set a backup password!
